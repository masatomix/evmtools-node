# テスト標準

[目的: 何をテストするか、テストの置き場所、構造を統一する]

## 方針
- 実装ではなく振る舞いをテストする
- 速く信頼性の高いテストを優先し、壊れやすいモックを最小化する
- クリティカルパスを深くカバーする。100% を追わず広さを取る

## 配置
選択肢:
- 同居型: `component.tsx` + `component.test.tsx`
- 分離型: `/src/...` と `/tests/...`
既定を 1 つ選び、例外は根拠付きで許容する。

命名:
- ファイル: `*.test.*` または `*.spec.*`
- スイート: テスト対象が何か。ケース: 期待される振る舞い

## テスト種別
- ユニット: 単一ユニット、依存はモック、超高速
- 統合: 複数ユニットを組み合わせ、外部のみモック
- E2E: フル経路、モック最小限、重要ジャーニーのみ

## 構造（AAA）
```typescript
it('does X when Y', () => {
  // Arrange
  const input = setup();
  // Act
  const result = act(input);
  // Assert
  expect(result).toEqual(expected);
});
```

## モックとデータ
- 外部はモック（API/DB）、テスト対象はモックしない
- ファクトリ/フィクスチャを使う、テスト間で状態をリセットする
- テストデータは最小限かつ意図が読み取れるように

## カバレッジ
- 目標: [全体 %]、重要ドメインはより高く
- CI で閾値を強制、例外はレビューで根拠が必要
