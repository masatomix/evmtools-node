# デプロイ標準

[目的: 環境・パイプラインの明確なパターンで、安全かつ再現可能なリリースを行う]

## 方針
- 自動化する。デプロイ前にテスト、デプロイ後に検証する
- 高速にロールバックできる段階的ロールアウトを優先する
- 本番変更は観測可能で取り消し可能であること

## 環境
- Dev: 高速イテレーション、デバッグ有効
- Staging: 本番をミラー、リリース検証用
- Prod: 強化済み、監視、最小権限

## CI/CD フロー
```
Code → Test → Build → Scan → Deploy (staged) → Verify
```
原則:
- テスト／スキャンで fail fast、デプロイをブロック
- アーティファクト・ビルドは再現可能（lockfile、固定バージョン）
- 本番は手動承認、監査ログを残す

## デプロイ戦略
- Rolling: 段階的にインスタンスを置き換える
- Blue-Green: 2 プール間でトラフィックを切り替える
- Canary: 少数ユーザーから開始、健全なら拡大する
リスクプロファイルに応じて選択、既定を明記する。

## ゼロダウンタイムとマイグレーション
- ヘルスチェックがトラフィックをゲート、graceful shutdown を行う
- ロールアウト中は後方互換な DB 変更
- マイグレーションを別ステップに分離、ロールバック経路をテスト

## ロールバック
- 前バージョンを準備、リバートを自動化する
- fix-forward より速くロールバック、発動条件を明文化

## 設定とシークレット
- 12-factor、設定は環境変数経由、シークレットは絶対に commit しない
- シークレットマネージャ、ローテーション、最小権限、アクセス監査
- 起動時に必須環境変数を検証

## ヘルスとモニタリング
- エンドポイント: `/health`, `/health/live`, `/health/ready`
- 監視: レイテンシ、エラー率、スループット、飽和度
- SLO 違反／スパイクでアラート、疲労しない閾値に調整

## インシデント対応と DR
- 標準プレイブック: 検知 → 評価 → 緩和 → 通知 → 解消 → ポストモーテム
- リテンション付きバックアップ、リストアテスト、RPO/RTO の定義

---
_ロールアウトパターンと安全策に集中する。プロバイダ固有の手順は含めない。_
