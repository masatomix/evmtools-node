# プロダクト概要

evmtools-node は、Excel 形式のプロジェクト進捗管理ファイルを読み込み、EVM（アーンドバリューマネジメント）分析によってプロジェクトの健全性を可視化する TypeScript ライブラリである。チームは既存の Excel 運用を変えることなく、定量的なプロジェクト分析が可能になる。

## コアケーパビリティ

1. **Excel/CSV パース & プロジェクト再構築** — 構造化された Excel 進捗管理ファイルからタスク階層・担当・日程を読み取り、ドメインモデルとして再構築する
2. **EVM メトリクス算出** — PV（計画価値）、EV（出来高）、SPI（スケジュール効率指標）をタスク単位・プロジェクト集計単位で計算し、進捗の実態を数値化する
3. **スナップショット差分分析** — 2 時点のプロジェクト状態を比較し、タスクの追加・変更・削除、PV/EV の変動を可視化する
4. **要員別分析 & 完了予測** — 担当者ごとの作業量・遅延状況を集計し、SPI ベースの完了予測を提供する
5. **CLI & ライブラリ API** — yargs ベースの CLI コマンド群と、レイヤー別の npm exports によるプログラマティックな利用の両方をサポートする

## 想定ユースケース

- **定期進捗レポート**: 基準日を指定して PV/EV/SPI を算出し、プロジェクトの健全性を把握する
- **前回比較**: 前回スナップショットとの差分から「何が動いたか」を特定し、遅延やスコープ変動を検知する
- **要員管理**: 担当者別の負荷と進捗を分析し、リソース配分を最適化する
- **完了見通し**: SPI オーバーライドやシナリオ分析で、現実的な完了時期を予測する

## バリュープロポジション

Excel 運用を壊さずに EVM 分析を導入できる点が最大の価値。プロジェクトマネージャーは慣れた Excel に進捗を記録し続けるだけで、計画と実績の乖離、スケジュール遅延の兆候、要員別の負荷偏りを定量的に把握できる。

## ドメイン用語（必須知識）

| 用語 | 意味 |
|------|------|
| **Project** | タスクツリー・基準日・休日カレンダー・メタデータを持つ集約ルート |
| **TaskRow** | フラットなタスクレコード。日程・実績・EVM メトリクス算出メソッドを持つ |
| **TaskNode** | TaskRow をラップした階層ツリーノード（親子関係の走査用） |
| **BaseDate（基準日）** | EVM 評価の基準となる営業日終了時点の日付 |
| **PV / EV / SPI** | 計画価値 / 出来高 / スケジュール効率指標（EV÷PV、1.0 超で前倒し） |
| **PlotMap** | 稼働予定日を示す Boolean 配列（Excel シリアル値がキー） |
| **ExcludedTasks** | 日付不備・PlotMap 欠損等で計算対象外となったタスク群 |
| **ValidStatus / InvalidReason** | タスクの計算可否判定と除外理由の分類 |
| **SPI Override** | 完了予測時に SPI を手動補正する仕組み（実績 SPI が信頼できない場合に使用） |

詳細は [docs/GLOSSARY.md](../../docs/GLOSSARY.md) を参照。
