# レビューチェックリスト

**作成日**: 2025-12-22
**用途**: PRレビュー時にcode-reviewerエージェントが参照

---

## 概要

このチェックリストは、PRレビュー時に確認すべき項目をまとめたものです。
`code-reviewer` エージェントはこのリストに基づいてレビューを実施します。

---

## 1. 必須チェック項目

### 1.1 自動チェック（CI）

| 項目 | 確認方法 |
|------|---------|
| lint PASS | `npm run lint` |
| format PASS | `npm run format` |
| test PASS | `npm test` |
| build PASS | `npm run build` |

### 1.2 コード品質

| 項目 | 確認観点 |
|------|---------|
| コーディング標準準拠 | `docs/standards/CODING_STANDARDS.md` に準拠しているか |
| 命名規則 | クラス/関数/変数名が規則に従っているか |
| アーキテクチャ準拠 | 依存関係ルール（domain は他層に依存しない）を守っているか |
| 不要なコード | 未使用のimport、変数、関数がないか |
| 不要なコメント | 古いコメント、自明なコメントがないか |

### 1.3 セキュリティ

| 項目 | 確認観点 |
|------|---------|
| 入力検証 | 外部からの入力を適切に検証しているか |
| 機密情報 | パスワード、APIキーなどがハードコードされていないか |
| OWASP Top 10 | 一般的な脆弱性パターンがないか |

---

## 2. ドキュメントチェック項目

### 2.1 仕様駆動開発

| 項目 | 確認観点 | 該当条件 |
|------|---------|---------|
| 要件定義書 | `REQ-xxx.md` が存在するか | 新機能追加時 |
| 詳細仕様書 | `{Class}.spec.md` が存在するか | 新機能追加時 |
| マスター設計書更新 | `{Class}.spec.md` に反映されているか | 既存クラスへの機能追加時 |
| トレーサビリティ | 要件→仕様→テスト→実装のリンクが正しいか | 新機能追加時 |

### 2.2 受け入れ基準

| 項目 | 確認観点 |
|------|---------|
| テスト証跡 | 要件定義書の受け入れ基準にテスト結果が記載されているか |
| ステータス更新 | 要件定義書のステータスが適切に更新されているか |

---

## 3. レビュー判定基準

### Approve

以下のすべてを満たす場合：
- 必須チェック項目に問題がない
- セキュリティ上の懸念がない
- ドキュメントが適切に更新されている

### Request Changes

以下のいずれかに該当する場合：
- セキュリティ上の問題がある
- テストがPASSしていない
- 必須のドキュメント更新が漏れている
- アーキテクチャ違反がある

---

## 4. チェックリスト（コピー用）

PRレビュー時にコピーして使用：

```markdown
## レビューチェック

### 自動チェック
- [ ] lint PASS
- [ ] format PASS
- [ ] test PASS
- [ ] build PASS

### コード品質
- [ ] コーディング標準に準拠
- [ ] 命名規則に準拠
- [ ] アーキテクチャ準拠（依存関係ルール）
- [ ] 不要なコード・コメントなし

### セキュリティ
- [ ] 入力検証が適切
- [ ] 機密情報がハードコードされていない

### ドキュメント（該当する場合）
- [ ] 要件定義書が存在
- [ ] 詳細仕様書が存在
- [ ] マスター設計書が更新されている
- [ ] トレーサビリティが更新されている
```
